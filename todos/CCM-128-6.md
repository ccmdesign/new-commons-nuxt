# [P2] ncResourceCard.vue uses computed without explicit import -- inconsistent with ncInitiativeCard

## File
components/ncResourceCard.vue

## Issue
`ncResourceCard.vue` uses `computed()` on line 32 without importing it from `'vue'`. This works via Nuxt auto-imports, but is inconsistent with the approach in the same PR: `ncInitiativeCard.vue` explicitly imports `{ resolveComponent, computed } from 'vue'`. Other existing components in the codebase (`ncTimeline.vue`, `ncCookieConsent.vue`) also use explicit imports. The inconsistency in a single PR makes the import strategy unclear.

## Suggested Fix
Add `import { computed } from 'vue'` to `ncResourceCard.vue` for consistency with the other components in this PR and the wider codebase. Alternatively, remove the explicit import from `ncInitiativeCard.vue` and rely on auto-imports everywhere (but then `resolveComponent` would still need an import).
