# [P2] `the-2025-challenge.vue` uses `el="a"` with `:href` for Rules button instead of NuxtLink pattern

## File
pages/the-2025-challenge.vue

## Issue
The "About the Challenge" section adds an `<nc-button el="a" ... :href="rulesUrl" target="_blank">` for the Rules link. While this works (external link, opens in new tab), the rest of the PR migrated away from `el="a"` to using the auto-detection via `to` prop for internal links. For external links, the correct pattern per the ncButton refactor is to NOT pass `el="a"` and instead use `href` directly (since `defaultEl` auto-detects `'a'` when `attrs.href` is present). The explicit `el="a"` is redundant but harmless.

## Suggested Fix
Remove the explicit `el="a"` from the Rules button -- `defaultEl` will auto-detect `'a'` from the `href` attribute. This keeps the codebase consistent with the new pattern.
