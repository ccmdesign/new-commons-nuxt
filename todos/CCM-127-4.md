# [P2] resources/index.vue missing `ref` and `computed` imports -- relies on Nuxt auto-import

## File
pages/resources/index.vue

## Issue
The script uses `ref(null)` and `computed(...)` without explicit imports. While Nuxt auto-imports these from Vue, the page also uses `useHead`, `useSeoMeta`, `useAsyncData`, and `queryCollection` without imports. This is consistent with the project pattern (Nuxt auto-imports), so it will work -- but unlike other pages in the codebase, this page has no `import` statements at all, which reduces clarity for developers unfamiliar with Nuxt auto-imports. Low risk but noting for consistency.

## Suggested Fix
No action required if the team is comfortable with Nuxt auto-imports. Optionally add `import { ref, computed } from 'vue'` for explicitness.
