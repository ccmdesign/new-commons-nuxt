# [P2] the-2025-challenge.vue nc-button removed el="a" prop for rules link -- relies on implicit attr detection

## File
pages/the-2025-challenge.vue

## Issue
The PR removes the `el="a"` prop from the `<nc-button>` that links to the rules Google Doc, leaving only `:href="rulesUrl"` and `target="_blank"`. The `ncButton` component's `defaultEl` computed will auto-detect `'a'` from `attrs.href`, so this technically works. However, the previous explicit `el="a"` was clearer in intent. More importantly, this button no longer has `rel="noopener noreferrer"` which is a security best practice for external links opened with `target="_blank"` (prevents `window.opener` attacks in older browsers).

## Suggested Fix
Add `rel="noopener noreferrer"` to the `nc-button` for the rules link. Optionally restore `el="a"` for clarity.
