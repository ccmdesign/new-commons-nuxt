# [P1] ncResourceCard hover shadow applies to non-linked (div) cards -- unintended visual feedback

## File
components/ncResourceCard.vue

## Issue
The `.resource-card:hover` rule (line ~73) applies `box-shadow: 0 12px 24px rgba(0, 0, 0, 0.08)` to ALL cards on hover, including cards rendered as `<div>` (when `hasLink` is false). This gives the user a visual affordance that the card is clickable when it is not. By contrast, `ncInitiativeCard.vue` correctly scopes its hover shadow to `a.initiative-card:hover`, applying it only when the card is a link.

## Suggested Fix
Change `.resource-card:hover` to `a.resource-card:hover` to match the `ncInitiativeCard` pattern and only show the hover shadow when the card is actually a link.
