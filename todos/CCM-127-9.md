# [P2] `useSiteLinks.js` composable has a TODO for updating rulesUrl but no mechanism to make it configurable

## File
composables/useSiteLinks.js

## Issue
The composable hardcodes a Google Docs URL and has `// TODO: Update rulesUrl when client provides new rules document`. This is a hardcoded external URL buried in a composable that is only used in one place (`the-2025-challenge.vue`). If the intent is to have a single source of truth for site-wide links, the composable should also include other external links (like the Google Spreadsheet link previously in the FAQ). As it stands, it is an over-abstraction for a single constant.

## Suggested Fix
Either consolidate other external URLs into this composable for consistency, or inline the URL directly in `the-2025-challenge.vue` and remove the composable. The TODO comment should be resolved before merge or converted to a tracked issue.
