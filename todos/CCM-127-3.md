# [P1] Resource card renders nothing when resource link is '#' -- no fallback UI

## File
components/ncResourceCard.vue

## Issue
The entire card is wrapped in `v-if="content && getResourceLink(content) !== '#'"`. When a resource has neither `url` nor `file` set (both optional in the schema), `getResourceLink` returns `'#'` and the card renders **nothing at all** -- no error, no placeholder, just a missing card in the grid. This creates a confusing layout gap. Since `url` and `file` are both optional in `content.config.ts`, this scenario is likely with partially configured resources.

## Suggested Fix
Either: (a) render the card without a link wrapper (use a `<div>` instead of `<a>`) when there's no valid URL, or (b) make at least one of `url` or `file` required in the schema to prevent the issue at the data layer.
