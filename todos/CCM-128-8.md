# [P2] Homepage resources section may render empty grid if no resources exist in content collection

## File
pages/index.vue

## Issue
Lines ~73-80 add a resources section to the homepage that queries `queryCollection('resources').limit(3).all()`. If the resources collection is empty (which is likely since `content/` is generated from Directus at build time and may not have resources yet), the section will render an `<h2>Resources</h2>` heading with an empty `.grid` container below it. There is no `v-if` guard or empty state. This will look broken to users.

## Suggested Fix
Add `v-if="resources?.length"` to the `<nc-base-section id="resources-section">` element to hide the entire resources section when no resources are available. This matches the defensive pattern used elsewhere in the codebase.
