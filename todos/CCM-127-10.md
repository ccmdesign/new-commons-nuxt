# [P2] `directus/resources.js` has inconsistent `await` usage in `getResources` function

## File
directus/resources.js

## Issue
In the `getResources` function, the `else` branch (lines 44-47) calls `objectContructor(dir, fs)` without `await`, while the `if` branch (line 35) correctly uses `await objectContructor(dir, fs)`. This inconsistency could cause the process to exit before all resource files are written. Additionally, `objectContructor` is misspelled (missing an 's' in "Constructor") -- this matches the existing pattern in other importers but is still a latent typo.

## Suggested Fix
Add `await` before `objectContructor(dir, fs)` in the `else` branch for consistency. The typo matches the existing codebase convention so it can be left as-is.

resolved
