# [P2] index.vue initiative card uses `<div>` instead of `<a>` or `<NuxtLink>` for "Indigenous Languages"

## File
pages/index.vue

## Issue
The Initiatives section has two cards. The first ("The 2025 Challenge") uses `<NuxtLink to="/the-2025-challenge">` correctly. The second ("Indigenous Languages") uses a plain `<div>` with the same `.initiative-card` class but is not interactive. It has `cursor: pointer` implied by hover styles and a hover shadow effect, which will mislead users into thinking it is clickable when it is not. The TODO comment says the URL is pending, but the card should either be hidden until a URL is available or styled differently to indicate it is not interactive.

## Suggested Fix
Either: (a) remove the hover styles for non-linked initiative cards by using a separate class, (b) hide the card until the URL is available with a `v-if`, or (c) wrap it in a disabled/placeholder state that visually communicates it is coming soon.
