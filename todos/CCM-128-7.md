# [P2] indigenous-languages.vue uses explicit composable import path while other pages rely on auto-import

## File
pages/incubator/indigenous-languages.vue

## Issue
Line 110: `import useSteeringCommittee from '../../composables/useSteeringCommittee'` uses a relative path import. In contrast, `the-2025-challenge.vue` also has explicit imports (`import useJudges from '../composables/useJudges'`), but most other pages in the codebase rely on Nuxt auto-imports for composables. While the explicit imports work, the relative path `../../composables/` is fragile and will break if the file moves. Nuxt auto-imports composables from the `composables/` directory by default.

## Suggested Fix
Remove the explicit import and use Nuxt auto-import: replace `import useSteeringCommittee from '../../composables/useSteeringCommittee'` with `const steeringCommittee = useSteeringCommittee()` directly (Nuxt auto-imports default exports from `composables/`). Apply the same fix to `the-2025-challenge.vue` imports for consistency.
