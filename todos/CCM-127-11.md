# [P2] `content.config.ts` resources schema allows all fields optional -- no validation that resource has a link

## File
content.config.ts

## Issue
The resources schema defines `url`, `file`, and `cover_image` all as `z.string().optional()`. This means a resource can be created with just a title, slug, and description but no link (`url` or `file`) and no image. The `ncResourceCard` template hides the entire card when `getResourceLink` returns `'#'` (see CCM-127-3), but there is no schema-level validation to catch this early. A resource without any link target is invalid content.

## Suggested Fix
Add a Zod refinement to ensure at least one of `url` or `file` is provided, e.g., `.refine(data => data.url || data.file, { message: 'Resource must have either url or file' })`. Alternatively, document this constraint in the schema with comments.
