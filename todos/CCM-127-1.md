# [P1] NuxtLink uses `href` instead of `to` in index.vue -- broken client-side routing

## File
pages/index.vue

## Issue
Line 46 uses `<NuxtLink href="faq" ...>` instead of `<NuxtLink to="/faq" ...>`. In Nuxt 3, `NuxtLink` requires the `to` prop for client-side routing. Using `href` falls back to a regular anchor element behavior, causing a full page reload and a relative URL (`faq` instead of `/faq`) which may resolve incorrectly depending on the current route (e.g., navigating from `/incubator/2026` would go to `/incubator/faq` instead of `/faq`).

## Suggested Fix
Change `<NuxtLink href="faq" ...>` to `<NuxtLink to="/faq" ...>` (note the leading slash for an absolute path).
